# Chat Server - 基于HTTP和WebSocket的聊天服务器

## 项目简介
本项目是一个轻量级的聊天服务器，同时支持HTTP协议和WebSocket协议，能够为客户端提供网页访问入口，并通过WebSocket实现实时的双向通信，支持多线程部署以提升并发处理能力。

## 环境依赖
- 操作系统：Linux
- 编译工具：CMake (>= 3.10)、GCC/G++ (>= 7.0) 
- 依赖库：
  - libwebsockets（WebSocket协议处理）
  - pthread（多线程支持）
  - 标准C/C++库（C++11及以上）

## 编译与运行
### 1. 编译步骤
```bash
# 创建编译目录并进入
mkdir build && cd build

# 生成Makefile
cmake ..

# 编译项目
make
```

### 2. 启动服务器
```bash
# 运行服务器程序
# 参数说明：0.0.0.0（监听所有网卡） 8080（端口号） n（线程数，n为自定义数字，如4、8等）
./chat_server 0.0.0.0 8080 n
```
示例（启动4线程的聊天服务器）：
```bash
./chat_server 0.0.0.0 8080 4
```

## 测试方式
1. 服务器启动成功后，打开本地浏览器；
2. 在地址栏输入：`http://localhost:8080`；
3. 即可进入聊天页面，支持多客户端（多个浏览器标签/窗口）连接并实时聊天。

## 核心功能
- ✅ HTTP服务：提供聊天页面的静态资源访问（如HTML/CSS/JS）；
- ✅ WebSocket通信：实现客户端与服务器、客户端之间的实时消息推送；
- ✅ 多线程支持：通过指定线程数，提升服务器并发处理能力；
- ✅ 跨客户端聊天：多个浏览器客户端连接后可互发消息。

## 架构说明
- **HTTP模块**：处理客户端的页面请求，返回聊天界面的静态资源；
- **WebSocket模块**：建立客户端与服务器的长连接，负责实时消息的接收、转发；
- **线程池模块**：根据启动参数创建指定数量的工作线程，分摊连接处理压力，提升并发效率。

## 常见问题
1. **端口被占用**：若8080端口被占用，启动时可替换为其他端口（如8081），浏览器访问对应端口即可；
2. **编译失败**：检查CMake、编译器版本是否满足要求，确认libwebsockets库已正确安装；
3. **客户端无法连接**：确认服务器监听地址为`0.0.0.0`（允许外部访问），防火墙已放行对应端口。

## 扩展方向
- 增加用户认证功能（如用户名/密码登录）；
- 支持私聊、房间/频道功能；
- 增加消息持久化（存储聊天记录到文件/数据库）；
- 支持WebSocket心跳检测，避免连接异常断开。
